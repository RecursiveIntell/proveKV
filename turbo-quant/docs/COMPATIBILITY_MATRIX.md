# Compatibility matrix

| Project | Relationship to turbo-quant | Allowed integration | Forbidden integration |
|---|---|---|---|
| semantic-memory | canonical vector/retrieval owner | compressed sidecar, candidate generation, exact rerank, receipts | replacing exact vector authority |
| Gloss | user-facing notebook/RAG product | shadow-mode demo with drift and top-k overlap | silent compressed retrieval |
| Recall | runtime memory/assistant layer | receipt-bearing derived memory acceleration | hidden truth store or unreceipted fallback |
| ClaimLedger | claim/evidence tracker | benchmark claims and receipts | storing marketing claims as facts |
| AiDENs | future orchestration/governor | policy selection and fallback through receipts | codec-specific hidden control plane |
| fib-quant | sibling algorithm crate | shared schemas, profile names, benchmark receipt compatibility | merging source into turbo-quant |
| AgentSecurity | scanner/admission layer | static review of generated codec code and dependencies | network/runtime assumptions without evidence |
